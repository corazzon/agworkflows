---
description: 3. 보고서 작성 (Comprehensive Reporting)
---
// turbo-all

1. **[보고서 기획]** 분석된 데이터를 바탕으로 스토리라인을 구성합니다.
    - **타겟 독자**: 비즈니스 의사결정권자 또는 일반 사용자.
    - **핵심 메시지**: 데이터에서 발견된 가장 중요한 인사이트 3가지 이상.

2. **[마크다운 보고서 작성]** `results/report.md` (또는 유사한 경로)에 상세 보고서를 작성합니다.
    - **필수 요소**:
        - **요약(Executive Summary)**: 전체 내용을 3줄로 요약.
        - **상세 분석**: 각 분석 주제별로 그래프와 설명을 배치.
        - **데이터 테이블**: 그래프의 근거가 되는 데이터를 마크다운 표로 제시.
        - **인포그래픽**: Mermaid 다이어그램이나 구조화된 텍스트 박스([!TIP], [!IMPORTANT] 등)를 활용하여 시각적 효과 극대화.

3. **[웹 보고서 변환 (HTML)]** 마크다운 보고서를 웹 브라우저에서 보기 편한 HTML 형식으로 변환하거나, Streamlit을 활용해 정적 리포트 페이지를 생성합니다.
    - `pandoc`이나 파이썬 마크다운 라이브러리를 활용하여 스타일이 적용된 HTML 파일 생성.
    - 혹은 `streamlit`의 `st.markdown`을 활용하여 미려한 UI로 렌더링.

4. **[발표 자료 변환 (PPTX)]** 핵심 내용을 프레젠테이션 슬라이드로 변환합니다.
    - 파이썬 `python-pptx` 라이브러리를 활용하여 주요 그래프와 요약 텍스트를 슬라이드 장표로 자동 생성.
    - **레이아웃**: 제목, 본문(텍스트+그래프), 결론이 포함된 템플릿 적용.

5. **[최종 검토]**
    - 생성된 HTML 및 PPTX 파일이 마크다운 원본의 내용을 충실히 반영하는지 확인.
    - 그래프의 가독성(폰트 크기, 색상)과 표의 정렬 상태 점검.

6. **[안내]** "보고서 생성이 완료되었습니다. `results/` 폴더 내의 HTML 및 PPTX 파일을 확인하세요."라고 안내합니다.
